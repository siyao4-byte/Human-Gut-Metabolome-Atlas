# Figure 4. Atlas-derived libraries enable faecal dataset remining and network interpretation

**a,** Platform-matched MZmine reprocessing of crude atlas datasets using atlas-derived in-house MSP libraries, compared with corresponding MAPS outputs. Stacks show distinct identities by annotation source. **b,** Public faecal dataset remining with atlas-derived MSP libraries. Labels show non-repeating annotations and annotation rate, calculated as non-repeating atlas-library annotations divided by resolved MZmine features. **c,** Identity overlap among the atlas, healthy-faecal HMDB records and MiMeDB with lipid-associated identities retained. Standardised compound name was used as the primary key and SMILES as fallback. **d,** NPClassifier superclass breakdown of atlas-only identities from the comparison in **c**. **e,** NIST faecal feature-based molecular network annotated with the atlas-derived MSP library. Node colour indicates atlas MSP match confidence, node size indicates subject prevalence, gold outer marks indicate atlas-matched identities absent from HMDB and MiMeDB, and edge width indicates MS/MS cosine similarity.

Atlas-derived libraries were generated for Phe-Hex, HILIC and SAX in positive- and negative-ion modes. MAPS annotations were restricted to confidence levels 1-3 and MSNovelist annotations were excluded. Public-dataset searches did not use retention-time matching and should be interpreted as spectral-library remining.
